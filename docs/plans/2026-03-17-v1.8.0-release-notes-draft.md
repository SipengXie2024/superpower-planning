# v1.8.0 Release Notes (Draft)

## Highlights

This release strengthens `superpower-planning` as a planning-first fork with stronger quality gates, clearer review semantics, and improved documentation of the persistent-memory workflow.

## Features

- Added a **spec review loop** to `brainstorming`
  - newly written specs now go through reviewer subagent feedback before planning
  - supports fix-and-rerun up to 3 rounds
  - adds an explicit user review gate before implementation planning
- Added a **plan review loop** to `writing-plans`
  - implementation plans now go through reviewer feedback before execution handoff
  - encourages lower-freedom, more repeatable review dispatch
- Strengthened **scope and decomposition discipline**
  - large requests should be decomposed before detailed design
  - one spec should not cover multiple independent subsystems
  - file boundaries and responsibilities are now emphasized earlier
- Strengthened **code review semantics**
  - `code-reviewer` now explicitly evaluates plan alignment, code quality, architecture, docs/standards, and issue severity
- Added standard review dispatch templates
  - `skills/subagent-driven/spec-review-dispatch-template.md`
  - `skills/writing-plans/plan-reviewer-prompt.md`

## Documentation Improvements

- Updated `README.md` to reflect the current workflow:
  - spec review loop
  - plan review loop
  - archive-aware persistent planning model
  - explicit execution routing
- Updated `/execute-plan` command guidance
  - requires a reviewed implementation plan before execution
  - clarifies when work should go back to planning instead of forcing execution
- Upgraded `requesting-review` documentation to match the stronger review model

## Notes

This release intentionally keeps the fork’s core differentiators intact:

- `.planning/` remains the persistent working-memory center
- `archive/` remains part of the long-term planning model
- explicit execution routing remains in place:
  - Subagent-Driven
  - Team-Driven
  - Parallel Session

## Suggested release title

`v1.8.0 — review-gated planning and stronger persistent workflow docs`

# v1.10.0 Release Notes

## Highlights

Introduces a **Stashing Workflow** (`/stash` and `/resume-stash`) to safely pause unfinished work, switch contexts, and resume later with integrity checks. This completes the working memory lifecycle:

- **Active**: Work in progress
- **Stashed**: Paused unfinished work
- **Archived**: Completed work

## New Features

- **New Command `/stash`**:
  - Saves current `.planning/` state (progress, findings, agent contexts) to `.planning/stash/`
  - Resets active workspace for a clean slate
  - Guards against stashing empty/meaningless state

- **New Command `/resume-stash`**:
  - Restores a stash back to active `.planning/`
  - **Freshness Check**: Explicitly verifies if findings are still valid or obsolete against current repo state
  - **Overwrite Guard**: Prevents accidental overwriting of existing active work (even if finished but unarchived)

- **Archive Reminder**:
  - `finishing-branch` now explicitly asks if you want to archive upon completion
  - Adds a persistent `ARCHIVE REMINDER` to dashboard if you choose to wait
